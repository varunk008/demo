Set Up a Multi-Node Apache Spark Cluster

This blog explains how to set up Apache Spark on a multi-node cluster. there I explain all the steps to install spark on a multi-node cluster.

Spark Architecture:-.

As you can see in this architecture we have one master node and two slave nodes. In the spark cluster, you can choose any number of masters and slaves. I have used one Master and two slaves.

Recommended Platform and Tools:-
Operating system - I have used ubuntu 21.10 os to set up the master machine and slave machines

Dependency - python3.9, java 1.8, and apache spark 3.2.1.

Installation process -
first, you have to create three clusters one as master and two as slaves in a virtual box.

Step1:- First check the IP address of the master and slaves.

~$CONTENT$nbsp;  ifconfig

Step2:-  configure hosts for all the machines to connect to each other.

~$ gedit  /etc/hosts

Step3:- check the connection using ping commands.

Step 4:-Now they ping each other and now disable the firewall for all the machines because we don't want any interruption while the data transfer in all the machines.

~$CONTENT$nbsp;  ufw disable

Step 5:- Now install the OpenSSH server on all the machines.

~$CONTENT$nbsp;  sudo apt-get install openssh-server

Step 6:- now we have to give root privileges to the user(in my case user is varun) on all the machines.

Note- please login as root using( sudo su)

Step 7:- Now it's time to install java (JDK 1.8). first, download the jdk file from https://www.oracle.com/in/java/technologies/javase/javase8-archive-downloads.html this link. extract the file and save the folder name as jdk.

Note:- Please use only jdk 1.8 otherwise spark will not work.

and the make directory jdk inside the /usr/lib and past all the files that you extract and the update .bashrc file. please do the same step in all the machines.

check the path is correct or not using the command (source .bashrc)

so first download the spark 3.2.1 from this link (https://spark.apache.org/downloads.html) and then extract the file and rename the file as a spark.

make one directory spark inside the /usr/local/ and past all the files that you extract.

step 9:- go to the conf folder inside the spark and configure these files.

log4j.properties
spark-defaults.conf
spark-env.sh

Step 10:- again edit .bashrc file.

Step 11:- spark need passwordless access in all the machine so we have to generate a public and private key in all the machines.

~$CONTENT$nbsp; ssh-keygen -t rsa -p ""

check whether keys are generated or not -

~$ ls -all .ssh/

check the same thing in other machines also.

Step12:- public and private keys are generated in all the machines. now we need to copy the public key from the master and share it with slave2 and slave2.

do the same step to slave1. copy the public key from the slave1 and share it with the master and slave2

do the same step to slave2. copy the public key from the slave2 and share it with the master and slave2

Step13:- Now copy the public key inside the authorized key so that it recognizes itself and allows passwordless access to itself.

do the same step in all the machines.

Step14:- Now start all the clusters.

~$ start -all.sh
